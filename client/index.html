BIENVENIDOS A LA API DE ENVIOS DE MARKETPLACE

Esta api es un microservicio de la api marketplace encargada del manejo de los envíos

Rutas de acceso y formato de la información

GET()

La ruta para leer los envíos es /envíos, tiene un paginado de 100 envíos por llamado, y estan ordenados de más recientes a más antiguos

El metodo GET devuelve un arreglo de objetos con la siguiente estructura

  • { data: [ {
  • idEnvio:number
  • idCliente: string,
  • idPedido: string,
  • estado: estadoEnum,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: boolean;
  • createdAt: Date
  • updatedAt: Date
  • } ] }

GET(:/id)

La ruta leer un envío especifico es /envíos/1, suponiendo que 1 sea la id del envío a buscar

El metodo GET(:/id) devuelve un objeto con la siguiente estructura

  • {
  • idEnvio:number
  • idCliente: string,
  • idPedido: string,
  • estado: estadoEnum,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: boolean;
  • createdAt: Date
  • updatedAt: Date
  • }

GET(:/id?estado=estadoEnum)

La ruta para cambiar el estado de un envío es /envíos/1?estado=pendiente, suponiendo que 1 sea la id del envío a cambiar y pendiente sea el estado al que se lo quiera cambiar.

Como parametro de la query hay que pasarle uno de los siguientes estados, estos estados son los mismos que devuelven en el campo estado cada uno de los objetos envíos.

  • pendiente
  • en transito
  • cancelado
  • entregado

El metodo GET(:/id?estado=estadoEnum) devuelve un objeto con la siguiente estructura, con el estado actualizado.

  • {
  • idEnvio:number
  • idCliente: string,
  • idPedido: string,
  • estado: estadoEnum,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: boolean;
  • createdAt: Date
  • updatedAt: Date
  • }

POST()

La ruta generar un nuevo envío es /envíos

En el body se le debe pasar la siguiente estructura:

  • {
  • idCliente: email,
  • idPedido: string,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: string,
  • }

Y devuelve un objeto con la siguiente estructura:

  • {
  • idEnvio:number
  • idCliente: string,
  • idPedido: string,
  • estado: estadoEnum,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: boolean;
  • createdAt: Date
  • updatedAt: Date
  • }

PUT(:/id)

La ruta generar un nuevo envío es /envíos/1, suponiendo que sea 1 la ide del envío que se quiere editar

En el body se le debe pasar la siguiente estructura:

  • {
  • idCliente: email,
  • idPedido: string,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: string,
  • }

Y devuelve un objeto con la siguiente estructura:

  • {
  • idEnvio:number
  • idCliente: string,
  • idPedido: string,
  • estado: estadoEnum,
  • tiempoEstimado: string,
  • direccionEnvio: string,
  • companiaTransporte: string,
  • numeroSeguimiento: boolean;
  • createdAt: Date
  • updatedAt: Date
  • }

Datos extras a tener encuenta

*** El objeto estadoEnum debe ser un string que puede tener solo estas palabras:

'pendiente', 'en transito', 'cancelado', 'entregado'

Estas se pasan por Query para cambiar el estado del envío y son las que vemos cuando llamamos un metodo get en cada objeto envío que leemos

*** Los cambio de estado de los pedidos desencadenan que se anuncie al cliente por email la situación de su pedido.

-- 'cancelado' envía el siguiente email:

Hola 'exemple@exemple.com' muchas gracias por confiar en nuestros servicios, te comentamos que tu pedido 'datos del pedido' ha sido cancelado. Que tenga un lindo dia

-- 'en tramite' envía el siguiente email:

Hola 'exemple@exemple.com' muchas gracias por confiar en nuestros servicios, te comentamos que tu pedido 'datos del pedido' se encuentra en tramite y llegara a la 'direccion de envio' en 'tiempo estimado de entrega'.

La compañía encargada del traslado es 'compania de traslado' y su número de seguimiento 'numero de seguimiento'. Si desea cancelar el pedido sin el reembolso puede hacerlo siguiendo este enlace

Cancelar pedido
